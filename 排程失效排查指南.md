# GitHub Actions 排程失效排查指南

> 記錄於 2026-04-04｜專案：川普每日摘要

---

## 問題描述

GitHub Actions 的 `schedule`（cron 排程）完全沒有自動執行，
但手動點擊 `workflow_dispatch` 觸發時卻正常運作。

---

## 根本原因

### 原因一：排程只會從「預設分支」執行

GitHub 的 `schedule` 觸發器**永遠只讀取預設分支（通常是 `master` 或 `main`）上的 workflow 檔案**，不會理會其他功能分支。

本次情況：

| 分支 | workflow 狀態 |
|------|--------------|
| `master`（預設分支） | 舊版，**缺少 `env` 區塊**，secrets 無法注入 |
| `claude/fix-workflow-schedule-jQKXh` | 新版，有完整 `env` 區塊 |

手動觸發時可以在 GitHub UI 選擇要用哪個分支執行，所以選到新分支就正常。
排程觸發時只會跑 `master`，跑到的是舊版、沒有 secrets，導致腳本失敗。

**結論：功能分支上的修復沒有合併回 `master`，排程就永遠看不到新版。**

---

### 原因二：缺少 `env` 區塊導致 Secrets 未注入

舊版 workflow（在 `master`）長這樣：

```yaml
jobs:
  send-digest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Run daily digest
        run: python trump_dailydigest.py
```

沒有 `env` 區塊，表示 `WEBHOOK_URL`、`MAIL_TO` 等 GitHub Secrets
**完全沒有被傳入**執行環境，腳本只能用寫死的預設值（可能已失效）。

修復後的版本加上了：

```yaml
jobs:
  send-digest:
    runs-on: ubuntu-latest
    timeout-minutes: 15

    env:
      WEBHOOK_URL: ${{ secrets.WEBHOOK_URL }}
      MAIL_TO: ${{ secrets.MAIL_TO }}
      CNN_ARCHIVE_URL: ${{ secrets.CNN_ARCHIVE_URL }}
      TRUTH_SOCIAL_ACCOUNT_ID: ${{ secrets.TRUTH_SOCIAL_ACCOUNT_ID }}
      TRUTH_SOCIAL_API_URL: ${{ secrets.TRUTH_SOCIAL_API_URL }}
      REQUEST_TIMEOUT: "30"
      RECENT_HOURS: "24"
      TRANSLATE_ENABLED: "1"
```

---

### 原因三：GitHub 會自動暫停閒置倉庫的排程

若一個倉庫**超過 60 天沒有任何 commit 活動**，GitHub 會自動停用排程 workflow。
停用後即使 cron 時間到了也不會執行，且不會發任何通知。

重新啟用方式：
1. 前往 `GitHub 倉庫 → Actions`
2. 左側點擊該 workflow 名稱
3. 若看到黃色警示橫幅，點擊 **"Enable workflow"**

---

## 修復步驟（本次實際執行）

1. 在功能分支 `claude/fix-workflow-schedule-jQKXh` 加入 `env` 區塊並修復 workflow
2. 將該分支推送至 GitHub：
   ```bash
   git push -u origin claude/fix-workflow-schedule-jQKXh
   ```
3. 在 GitHub 建立 Pull Request：`claude/fix-workflow-schedule-jQKXh` → `master`
4. Merge PR，讓 `master` 取得正確的 workflow
5. 至 Actions 頁面確認排程已啟用

## 日後預防清單

| 檢查項目 | 說明 |
|----------|------|
| 修改 workflow 後記得合併回 `master` | 排程只看預設分支 |
| workflow 有用到 secrets 就必須加 `env` 區塊 | 少了就等於沒設定 |
| 超過兩個月沒 commit，主動去 Actions 確認排程狀態 | 避免被 GitHub 自動暫停 |
| 手動觸發測試時，確認選的是 `master` 分支 | 才能真實模擬排程環境 |
